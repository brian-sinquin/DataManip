# Contributing to DataManip

Thank you for your interest in contributing to DataManip! We welcome contributions from the community.

## Getting Started

Before you begin:
- Check existing issues and pull requests to avoid duplicates
- Read through this guide to understand our workflow

## Coding Style

Please adhere to the following coding standards:
- Follow consistent naming conventions throughout the project
- Write clear, descriptive variable and function names
- Add comments for complex logic
- Keep functions small and focused on a single task
- Format code consistently (use provided linting rules if available)

## Project Architecture

Understanding the project structure will help you navigate the codebase:
- `/src` - Main source code
- `/docs` - Documentation files
- `/tests` - Test files and test data
- `/examples` - Usage examples

Please maintain this structure when adding new files.

## Building and Running

To build and run the project locally:

1. Clone the repository
2. Install dependencies (refer to README.md)
3. Run tests to ensure everything works
4. Make your changes
5. Test your changes thoroughly

## Pull Requests

To propose changes:

1. Fork the repository
2. Create a new branch (`git checkout -b feature/your-feature-name`)
3. Make your changes (optionnaly add your contributor informations in `assets/contributors.json`)
4. Commit with clear, descriptive messages
5. Push to your fork
6. Open a pull request with a detailed description

### PR Guidelines
- Link related issues in your PR description
- Ensure all tests pass
- Update documentation if needed
- Keep changes focused and atomic

## Reporting Issues

If you're not contributing code, you can still help:
- **Report bugs**: Open an issue with reproduction steps
- **Request features**: Describe the feature and its use case
- **Improve documentation**: Suggest clarifications or additions

Thank you for contributing!
